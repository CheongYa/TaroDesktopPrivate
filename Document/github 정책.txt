pull request를 바탕으로 작업을 한다.
1. 각 유저는 해당 레퍼지터리를 fork 하여 작업한다.
2. 충돌을 방지하기위해 각자 이름으로 파일을 만들어 그 안에서 작업한다.
3. 충돌을 방지하기위해 각자 작업하는 파일을 명시적으로 정하여 작업한다.
4. 충돌 가능성이 있는경우 파일을 나눠 작업한 후 인라인으로 처리하거나 직접 병합한다.
5. pull-request를 적극적으로 이용한다.
6. pull-request시 코드리뷰를 적극적으로 한다.
7. 커밋 메시지는 의미있게 작성한다.

main 브렌치	: 프로그램을 즉시 활용할 수 있는 형태
release 브렌치	: 프로그램의 통합작업을 하는 브렌치

메인 폴더	: 프로젝트에서 통합을 실행한다.
moudle 폴더	: dll, 사용자 정의 컨드롤 등을 저장, 관리한다.
document 폴더	: 프로젝트를 진행하며 생성된 모든 문서를 저장한다.
